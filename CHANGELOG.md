# CHANGELOG

本仕様書 (Reversi UART Protocol, RUP) のバージョンごとの変更履歴。

## v0.2 (ドラフト, 2026-05-XX)

**破壊的変更（v0.1 と非互換）**。実装者は移行が必要です。
v0.1 採用者がまだ少ない段階で、長期的に保守しやすい形へ刷新します。

### 主な変更

#### 1. 応答プレフィックス `+` / `-` を導入

POP3 (RFC 1939) の `+OK` / `-ERR` 形式に着想を得た 1 文字プレフィックスで
**「自発通知 vs 応答」「成功 vs エラー」** を文法レベルで明示する。

| プレフィックス | 意味 |
| ---- | ---- |
| `+` | ホスト直前送信への **成功応答** |
| `-` | ホスト直前送信への **エラー応答**（`<NN>` は §7.3 コード） |
| (なし) | **自発通知**（CPU 主導 / ゲーム進行通知） |

例:

```text
< VE                    PC: バージョン要求
> +VE01MyCPU-v1         CPU: 成功応答 (+ 付与)

< MOz9                  PC: 不正座標
> -04 bad coord          CPU: エラー応答 (- + コード)

> MOd3                  CPU: 自発の着手 (prefix なし)
> BS00...221...          CPU: 自発の盤面通知
> RE                    CPU: 自発のブート完了通知
```

**新ルール: 応答必須はクエリのみ**:

- クエリ系 (`PI`, `VE`, `X*`) → `+` または `-` 必須
- 通知系 (`SB`, `SW`, `MO`, `PA`, `BO`, `EB`, `EW`, `ED`) → 応答任意（CPU が
  返したいなら `+SB` 等を返してよい。PC は受信したら破棄）
- 仕様違反受信 → `-<NN> [reason]` 必須
- CPU 自発通知 → ホストからの応答なし

#### 2. `PO` mnemonic 廃止 → `+PI` に統一

```text
旧: < PI → > PO
新: < PI → > +PI
```

応答プレフィックス導入により、対になる別 mnemonic が不要に。

#### 3. `VE` 応答に `+` プレフィックス必須

```text
旧: < VE → > VE01MyCPU-v1
新: < VE → > +VE01MyCPU-v1
```

#### 4. `ER<NN>` mnemonic 廃止 → `-<NN>` に統一

```text
旧: > ER04 bad coord
新: > -04 bad coord
```

`-` プレフィックスがすでに「エラー」を意味するので `ER` mnemonic は冗長。

#### 5. `X*` 拡張ネームスペースを予約

CPU 製作者が独自コマンドを定義できる領域を新設。

- 大文字 2 文字で **1 文字目が `X`** のコマンド (`XA` 〜 `XZ`、26 スロット) は
  拡張領域として予約
- 将来の RUP 公式追加コマンドは **1 文字目を `A`-`W` に限定**（`X` は触らない）
- RUP 準拠 PC 側パーサは `X*` 受信を §5.1 違反としては扱わず、`-` も返さない
  （ログ記録のみ、状態機械にも影響しない）
- 応答書式は `+X*[<data>]` または `-<NN> [reason]`

例:

```text
< XB                   PC: 拡張ベンチマーク要求
> +XB clk=1.8MHz tick=557ns

< XI                   PC: 拡張デバイス情報要求
> +XI pf=rp2350-arm-s chip=B2 ram=520KB
```

#### 6. 全コマンドが 2 文字に統一

v0.1 期間中に検討されていた 3 文字拡張 (`XID`, `XBM` 等) は採用せず、
全コマンドを 2 文字に統一。

#### 7. 空行 (`\r\n` 単体) はサイレント破棄

§5.1 補足。コマンドバイトを 1 つも含まない `\r\n` 単体は仕様違反として扱わず、
ホスト・CPU ともに `-NN` を返さない。シリアル端末で Enter を誤打した場合や
行クリア目的の余分な `\r\n` を許容する。

#### 8. `CA` (CAPABILITY) コマンド追加（任意実装）

CPU が実装している受信可能コマンドを動的に問い合わせるクエリ。
POP3 (RFC 1939) `CAPA` / IMAP `CAPABILITY` 相当。

```text
< CA
> +CA PI VE CA SB SW MO PA BO EB EW ED XI XB
```

- 必須・任意・`X*` 拡張すべての mnemonic を空白区切りで列挙
- 順序は実装者自由
- `+CA` 自身も応答に含める（一貫性優先・パーサ単純化）
- 機械可読な機能一覧であり、人間向けヘルプテキストではない
- 任意実装（未実装 CPU は `-01` Unknown command を返す）
- 各 token は `[A-Z]{2}` を基本。将来 `MNEMONIC:<metadata>` 形式追加の余地を残し、
  パーサは未知の `:` 付き token を無視

#### 9. `X*` 拡張領域に推奨スロット表を追加（強制力なし）

§5 補足。実装者間のコンベンションとして `XI` (Identification) / `XB` (Benchmark) /
`XS` (Set Depth, NBoard `SD` 相当) / `XH` (Hint, NBoard `HI` 相当) を推奨割当として
明記。CPU 横断で同一 mnemonic に揃えると PC 側ツールが汎用化できる。
RUP 仕様としての強制力はない（あくまで `X*` 領域内の慣習）。

### 移行ガイド (v0.1 → v0.2)

CPU 側ファームウェアの変更点:

| 受信処理 | v0.1 | v0.2 |
| ---- | ---- | ---- |
| `PI` への応答 | `PO\r\n` 送信 | `+PI\r\n` 送信 |
| `VE` への応答 | `VE01name\r\n` 送信 | `+VE01name\r\n` 送信 |
| 仕様違反検知 | `ER04 ...\r\n` 送信 | `-04 ...\r\n` 送信 |
| 独自コマンド | （仕様外） | `X*` を選び、応答に `+`/`-` 付与 |

PC 側パーサの変更点:

- 受信した行が `+` で始まる → 直前送信したコマンドへの成功応答として処理
- `-` で始まる → エラー応答（`<NN>` を抽出）
- それ以外 → CPU 自発通知（`MO`/`BS`/`ST`/`NC`/`RE`/`RS`/`EN`/`PA`）として処理
- `X*` または `+X*` / `-` 受信は §5.1 違反扱いせず、ログのみ

### ドキュメント構成変更（仕様変更なし）

- PlantUML ソースおよび生成 PNG を `diagrams/` 配下へ集約
- 英語版図版を `<name>.png` → `<name>.en.png` に改名し、日本語版 `<name>.ja.png`
  と命名規則を対称化
- 各 `.puml` の `@startuml NAME` 行を裸の `@startuml` に統一（出力名を入力ファイル名から自動導出）
- 旧パスへの直リンクは破壊される。再生成は `cd diagrams && plantuml -tpng *.puml`

初版公開。tommieChat の実装と並行して仕様化。
